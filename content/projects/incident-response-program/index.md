# Enterprise Incident Response Program Design

## Overview

This project documents the end to end design of an Enterprise Incident Response Program and the supporting Incident Response Plan. The deliverable is structured as a portfolio artefact that demonstrates capability across IR governance, stakeholder coordination, communications planning, exercise design and operational metrics.

The work is presented as a standalone capability piece, not a coursework submission, and is sanitised for public review. All organisations, contacts and entities referenced inside the artefact are fictional or replaced with placeholders.

## Scenario Context

The engagement scenario places the author in the role of an internal lead responsible for planning and overseeing the development of an Incident Response Team for a client organisation. The task involves establishing a project team, evaluating the existing Incident Response Plan, and developing an updated Incident Response Plan. The key focus is a cyber security based project using current applications and preparing the organisation for Incident Response Team activities, with a selected online project management and collaboration platform supporting planning, coordination, communication and documentation throughout the lifecycle.

## What Was Delivered

The portfolio artefact contains the full set of design documents that make up the program:

- Incident Response Plan (IRP) with full lifecycle coverage and supporting annexes
- Communications Plan covering internal, external and other stakeholders
- Project Charter with scope, objectives, methodology, deliverables, timeline and budget
- Project Team Briefing Report covering team composition, roles, responsibilities and metrics
- Red, Blue and Purple Team exercise design and observer criteria
- Cyber Security Operations Centre (CSOC) recommendations and foundational guidance
- Annexes covering severity matrix, investigation questions, incident log and evidence register fields, exercise resources, situation report fields and a playbook summary with process map

## Key Capabilities Demonstrated

- Incident response lifecycle modelling across Prepare, Detect, Respond, Recover and Improve phases
- Alignment with NIST SP 800-61 and Australian Cyber Security Centre (ACSC) guidance
- Stakeholder communication planning across internal teams, external parties and regulators
- Operational metrics design, including Mean Time to Detect (MTTD) and Mean Time to Respond (MTTR)
- Tabletop and live exercise design across Red, Blue and Purple Team functions
- Project coordination across charter, scope, deliverables, timeline, budget and work breakdown
- CSOC capability planning aligned with monitoring, triage and response responsibilities
